# Coding Guide

## Core principle
Code each row from evidence in that specific time window, not from general impressions about the civilization.

## Time unit
Use 25-year windows unless the case requires a different window.

## Evidence rules
- Prefer at least 2 academic sources per row where possible.
- Use 9 for unknown or weak evidence.
- Record source_count and notes for each row.

## Scale
- 0 = absent / no evidence
- 1 = low
- 2 = moderate
- 3 = high
- 9 = unknown / insufficient evidence

## Outcome labels
- 0 = stable
- 1 = stressed
- 2 = severe decline
- 3 = collapse / fragmentation

## Good coding practice
- Code intensity, not morality.
- Separate drivers from downstream effects when possible.
- Do not force identical interpretations across ancient and modern cases.
- Keep notes short but explicit.

## Example note
"Repeated elite struggles, rising invasion pressure, fiscal weakness, and visible territorial loss."
